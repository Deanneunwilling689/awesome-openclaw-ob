# OpenClaw Foundation 治理

## 现状

- 维护者委员会正在组建中（GitHub: openclaw/maintainers 仓库已建立）
- 已知核心维护者中，仅 "Shadow" 一人在公开场景中被频繁提及
- [[Peter Steinberger]] 表示正在增加维护者并建立流程，目标是全职带薪维护
- 社区担忧：如果"基金会"变成"[[OpenAI 为什么收编 OpenClaw|OpenAI 控制]]"的代名词，社区将 fork
- 项目采用 MIT License，理论上任何人都可以 fork 并独立发展

## 与成熟开源治理的对比

| 维度 | Linux Foundation | OpenClaw Foundation |
|------|------------------|---------------------|
| 技术顾问委员会（TAB） | 成熟运作多年 | 尚未建立 |
| 全职维护者 | 数十人（企业赞助） | 正在招募，规模未知 |
| 治理文档 | 完善的章程和流程 | 仅有框架，细节待定 |
| 独立性保障 | 多元企业会员制衡 | OpenAI 为主要赞助者，制衡机制不明 |
| 安全响应团队 | 专职 | 无 |

## CloudBees 分析

CloudBees（2026.2.6）发表 "OpenClaw Is a Preview of Why Governance Matters More Than Ever"：
- AI Agent 正从"辅助"转向"执行"（即自主执行），治理成为关键区分因素
- 研究人员已发现 **42,000+ 个未保护的 OpenClaw 网关**暴露在互联网上
- 部署速度远超安全治理能力——这是整个 AI Agent 行业的缩影
- 治理不应是阻碍者（blocker），而应是控制平面（control plane）

治理空白是 [[OpenClaw 投资风险因素]] 的重要组成部分。
