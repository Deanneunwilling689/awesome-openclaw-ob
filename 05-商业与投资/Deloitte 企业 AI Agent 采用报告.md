# Deloitte 企业 AI Agent 采用报告

## 一句话总结

Deloitte 2025/2026 调查显示，仅 11% 的组织在生产环境部署了 AI Agent，38% 在试点阶段，42% 仍在制定策略，揭示了 Agent 技术的炒作与现实之间的巨大落差。

## 核心数据

| 指标 | 比例 |
|------|------|
| 生产环境部署 AI Agent | **11%** |
| 正在试点 Agent | **38%** |
| 仍在制定 Agent 策略 | **42%** |
| 完全没有 Agent 策略 | **35%** |
| 数据来源 | Deloitte 2025/2026 调查 |
| 相关报告 | Deloitte Tech Trends 2026 |

## 分析

Deloitte 的数据为 [[AI Agent 市场规模|AI Agent 市场]]的炒作降了温。在 Sam Altman 宣称"2025 年是 Agent 元年"、各大厂商争相发布 Agent 产品的同时，实际生产部署率仅有 11%。

这个数字与 [[Fortune 500 企业 AI Agent 部署|Microsoft Cyber Pulse 报告]]的"80%+ Fortune 500 部署活跃 AI Agent"形成有趣对比——差异在于定义：Microsoft 统计的是任何形式的 AI Agent 使用（包括 Copilot Studio 等轻量工具），而 Deloitte 关注的是**真正的生产环境部署**。

38% 的试点率说明企业不是不感兴趣，而是在安全、治理、ROI 等方面尚未跨过门槛。这与 [[OpenClaw 投资风险因素|OpenClaw 的安全债务]]和 [[NIST AI Agent 安全标准|NIST 启动标准制定]]的背景高度一致——企业需要标准和安全保障才敢大规模部署。

42% 仍在制定策略 + 35% 完全没有策略 = **超过 3/4 的组织尚未认真启动 Agent 部署**。

## 关键洞察

Deloitte 的数据是理解 [[AI Agent 市场规模|Agent 市场]]最重要的"清醒剂"之一。当 [[Gartner AI Agent 预测|Gartner 预测]] 2026 年底 40% 企业应用将集成 AI Agent 时，Deloitte 的 11% 生产部署率意味着需要在一年内实现约 **4 倍增长**——这不是不可能，但需要安全标准落地（[[NIST AI Agent 安全标准|NIST]]）、治理框架成熟（[[OpenClaw Foundation 治理|基金会化]]）、以及 [[OpenClaw 投资风险因素|安全事故]]频率下降。Karpathy 的判断可能更接近现实：Agent 的成熟需要的是一个"十年"，而非一年。
