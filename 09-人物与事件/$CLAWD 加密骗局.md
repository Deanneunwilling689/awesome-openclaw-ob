# $CLAWD 加密骗局

> **一句话总结**：Clawdbot 改名间隙中诈骗团伙在 10 秒内抢注旧社交账号，伪造"官方治理代币"$CLAWD，市值飙升至 $1600 万后崩盘归零，GitHub SVP 手动介入恢复被抢注账号。

## 核心要点

- 发生在 [[The Great Molt 改名事件]] 的改名间隙（2026.1.27-30）
- 诈骗团伙在 **10 秒内**抢注旧社交账号
- 伪造"官方治理代币"$CLAWD，市值一度飙至 **$1600 万**后崩盘归零
- [[Peter Steinberger]] 的 GitHub 用户名 **steipete** 被抢注，需 **GitHub SVP** 手动恢复

## 详细内容

| 短期内 | $CLAWD 市值飙升至 **$1600 万** |
| 最终 | 代币价格崩盘归零 |

### 骗局手法

1. **账号抢注**：在 Steinberger 释放旧社交账号的**几秒内**，机器人自动抢注
2. **身份伪造**：利用被抢注的官方账号发布虚假信息，冒充项目官方
3. **代币发行**：推出"$CLAWD"代币，声称为项目的"官方治理代币"
4. **FOMO 驱动**：利用项目改名的混乱和社区焦虑，制造紧迫感和购买压力
5. **"Handsome Molty"武器化**：将 AI 生成的恐怖人脸龙虾体图像作为 meme 宣传材料

### GitHub 账号事故

骗局的混乱中还发生了一个更严重的事故：

- Steinberger 在紧急改名过程中，意外改了**自己的 GitHub 个人账户名**而非组织名
- **steipete** 这个用户名在释放后被**立即抢注**
- 这意味着他失去了 GitHub 上的个人身份——所有历史提交、issue、PR 链接都指向了错误的人
- 需要 **GitHub SVP（高级副总裁）** 级别的介入才恢复了原用户名

### 骗局的社会背景

$CLAWD 骗局并非孤立事件。在加密货币领域，"名人代币"和"项目代币"骗局已形成产业化：

- 有专门的机器人监控热门项目的社交账号变更
- 改名/迁移是最容易被利用的窗口期
- 加密社区的 FOMO（害怕错过）心理被精准利用

### Steinberger 的情感代价

在 Lex Fridman 播客中，Steinberger 承认这是他最接近放弃的时刻：

> "I was close to crying"

他差点删除整个项目。这段经历也促使他在第二次改名（Moltbot → OpenClaw）时采取了"**曼哈顿计划**"级别的保密措施——使用诱饵名称、跨平台同步更换、提前致电 [[Sam Altman]] 确认新名称不侵权。

## 关键洞察

$CLAWD 骗局暴露了一个被忽视的安全维度：**数字身份迁移的攻击面**。当一个拥有数十万关注者的项目需要改名时，旧身份（社交账号、域名、包名）的释放窗口成为攻击者的黄金机会。10 秒内的抢注速度说明这类攻击已经高度自动化。GitHub SVP 的介入则反映了平台在面对此类攻击时，常规流程的无力——需要最高层级的人工干预。这个事件与 [[The Great Molt 改名事件]] 的其他灾难一起，为整个开源生态敲响了警钟：**品牌迁移需要像代码部署一样有预案和回滚机制**。[[Dario Amodei|Anthropic]] 的商标执法虽然合法，但未考虑到改名过程中的这些连锁安全风险，间接为骗子创造了机会。
