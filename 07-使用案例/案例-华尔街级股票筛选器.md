# 案例：华尔街级股票筛选器

**一句话总结**：一个人用 OpenClaw + Claude Opus 4.5 + 完全免费的数据源，花 $5/月 VPS 费用，构建了一个能扫描全部标普 500 股票、计算巴菲特 10 个完整质量公式的专业级股票筛选系统——以前这种工具只有机构投资者用得起。

**人物**：florinelchis，Medium 博主
**案例**：用 OpenClaw + Claude Opus 4.5 构建专业级股票筛选系统

## 系统能力

这个系统的能力覆盖了专业投资分析的多个层面——这个案例展示了 [[Tool Use 机制]] 的真正威力，Agent 自主调用金融 API 完成专业分析，将机构级能力交到个人手中：

### 技术分析层
- 扫描 **500 支标普 500 股票**的技术超卖信号
- RSI、MACD、布林带等经典技术指标的实时计算
- 自动识别潜在的买入时机

### 基本面分析层
- 巴菲特 **10 个质量公式**从 SEC 官方文件直接计算
- **非简化比率**——使用完整公式，而非财经网站常见的简化版本
- **来源追溯**：每个数据点都可以追溯到 SEC EDGAR 的原始文件编号
- 这意味着分析结果的可信度与专业投资机构使用的数据处于同一水平

### 交互界面
- **Telegram 命令操控**，三个简单命令：
  - `oversold`（15-30 秒）——扫描所有超卖股票
  - `screen`（1-3 分钟）——运行完整的巴菲特质量筛选
  - `analyze AAPL`（2-3 秒）——深度分析单支股票

## 成本结构

这是这个案例最令人震惊的部分：

| 项目 | 成本 |
|------|------|
| 数据源（Yahoo Finance + SEC EDGAR） | **$0**（100% 免费 API） |
| VPS 服务器 | **$5/月** |
| API 调用费（Claude Opus 4.5） | 按量计费（通常 $10-30/月） |
| **总计** | **$15-35/月** |

作为对比，专业级的 Bloomberg Terminal 费用是 **$24,000/年**（$2,000/月），FactSet 是 **$12,000/年**。这个 OpenClaw 方案的成本**不到 Bloomberg 的 2%**。

当然，功能覆盖面远不如 Bloomberg——但对于个人投资者做基本面筛选和技术分析来说，核心功能已经足够。

## 技术架构

### 为什么选择 Telegram？

- **无需网页仪表盘**：不用开浏览器，不用登录，不用坐在电脑前
- **移动优先**：在手机上随时随地查看分析结果
- **命令式交互**：三个命令覆盖 90% 的使用场景，学习成本为零
- 这与 [[案例-Telegram 聊天开发 iOS 应用]] 的设计理念一脉相承——**消息应用正在成为 [[AI Agent 定义与分类|Agent]] 交互的主流入口**，体现了 多频道消息架构 的实际价值

### 数据源选择

**Yahoo Finance**：
- 免费提供实时和历史价格数据
- 基础财务数据（收入、利润、资产负债表摘要）
- 有速率限制但对个人使用足够

**SEC EDGAR**：
- 美国证券交易委员会的官方文件数据库
- 所有上市公司的完整财务报告（10-K、10-Q、8-K）
- 100% 免费且无速率限制
- 数据权威性最高——这就是机构投资者使用的同一数据源

两者结合，覆盖了技术分析（价格数据）和基本面分析（财务数据）的完整需求。

## 核心洞察

### 工具平权：从机构特权到个人可及

这个案例的深层意义在于**能力的民主化**：

| 维度 | 以前 | 现在 |
|------|------|------|
| **数据获取** | Bloomberg $24K/年 | Yahoo Finance + SEC EDGAR $0 |
| **分析能力** | CFA 分析师 $100K+/年薪 | Claude Opus 4.5 按量计费 |
| **基础设施** | 机构级服务器 | $5/月 VPS |
| **交互界面** | 专业终端 | Telegram 三个命令 |

20 年前，能做到这些的只有华尔街的基金公司。10 年前，你需要自学 Python + 量化分析才行。现在，一个懂得描述需求的人 + Agentic Coding 就够了。这正是 [[编程民主化]] 和 可及性突破 的缩影，也验证了 非技术用户真实案例 中展现的趋势——大语言模型降低了专业工具的使用门槛。

### API 路线 vs 浏览器路线

与 [[案例-Tesco 超市自动购物]] 的纯浏览器方案不同，股票筛选器走的是 **API 集成路线**。两种路线各有优劣：

| 维度 | API 路线（本案例） | 浏览器路线（Tesco） |
|------|------------------|-------------------|
| **速度** | 快（直接数据传输） | 慢（页面渲染 + 操作） |
| **稳定性** | 高（API 接口稳定） | 低（网站改版即失效） |
| **适用前提** | 需要有免费/付费 API | 只需要有网页界面 |
| **适用场景** | 数据密集型任务 | 操作密集型任务 |

对金融数据来说，API 路线明显更优——需要处理 500 支股票的大量数据，浏览器逐个爬取会慢到不可用。

### 巴菲特公式的"完整实现"

值得特别强调的是"非简化比率，完整公式含来源追溯"这一点。多数财经网站和个人工具使用的是简化后的财务比率，可能丢失重要的细节。这个系统直接从 SEC 原始文件计算，确保了分析的严谨性。

这也是 AI Agent 的一个独特优势：它可以处理枯燥、繁琐、需要大量数据处理的工作，而不会偷懒走捷径。人类分析师面对 500 支股票 × 10 个公式可能会选择简化，Agent 不会。这种可靠性源自 Agent Execution Loop 的设计——每一步都严格执行，不受疲劳或情绪影响。结合 记忆系统 的持久化能力，Agent 还能随时间积累分析经验。
