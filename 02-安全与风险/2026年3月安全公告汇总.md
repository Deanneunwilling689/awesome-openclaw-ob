# 2026年3月安全公告汇总

## 严重度分布

- **严重（Critical）**：1 个 — CVSS 10.0 满分的 WebSocket 提权
- **高危（High）**：4 个 — 涉及 DoS、提权、RCE、越权访问
- **中危（Moderate）**：5 个 — 涉及信息泄露、路径注入、授权绕过

## 漏洞分类分析

### 认证与权限缺陷
- GHSA-rqpp：共享密钥连接可自行声明 `operator.admin`，与 [[权限控制机制]] 中已知的权限作用域问题一脉相承
- GHSA-63f5：bootstrap 令牌可反复重放提升权限
- GHSA-r7vr：命令授权用户越权访问 Owner-only 接口
- GHSA-7h7g：配对载荷嵌入永久凭证，与 [[凭证泄露与信息窃取]] 中的明文凭证问题同类

### 供应链与代码执行
- GHSA-99qw：`.openclaw/extensions/` 下的插件自动加载无需用户确认，是 [[恶意 Skills 供应链攻击]] 在本地工作区的延伸

### 平台集成安全
- GHSA-jq3f：Telegram Webhook 验证前即读取并缓冲请求体，造成资源耗尽
- GHSA-xwcj：Telegram 媒体获取错误信息中泄露 Bot Token
- GHSA-g2f6：iMessage SCP 操作数中未验证 shell 元字符
- GHSA-f5mf：Zalo 白名单使用可变群组名称而非稳定 ID

### 文件权限
- GHSA-vr7j：JSONL 会话记录文件权限未设为 `0o600`

## 其他安全改进（同期）

除上述 GHSA 公告外，2026年3月还包含以下安全修复：

- **SQL LIKE 通配符注入修复** — 防止通过 LIKE 查询的通配符进行信息探测
- **Ruby 审批流程代码加载标志默认拒绝** — 将默认行为从允许改为拒绝
- **Unicode 混淆攻击防护** — 命令审批提示中防止 Unicode 字符混淆
- **exec approval 欺骗修复** — 防止绕过执行审批流程
- **网络 SSRF 风险修复** — 限制服务端请求伪造攻击面

## 趋势观察

3月的安全公告反映了几个显著趋势：

1. **攻击面扩大**：从核心 WebSocket 到 Telegram、iMessage、Zalo 等多平台集成，攻击面随生态扩张而急剧扩大
2. **配对机制脆弱**：多个漏洞（#3、#10）指向配对/引导流程的设计缺陷，详见 [[配对机制安全缺陷]]
3. **权限模型持续受压**：`operator.admin` 提权路径仍是高价值攻击目标，印证了 [[致命三合一安全矛盾]]
4. **供应链风险本地化**：GHSA-99qw 表明供应链攻击已从 ClawHub 延伸到本地工作区
