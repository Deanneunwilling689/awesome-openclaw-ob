# 开源AI运动

## 一句话解释

想象AI模型是一道菜的配方——闭源意味着餐厅只卖成品不给食谱，开源意味着把配方公开，任何人都能做、都能改、都能卖。

## 核心理念

开源AI的本质诉求是：**AI的力量不应该被几家公司垄断**。当模型权重、训练代码、数据集全部公开时，全球开发者可以审计安全性、复现研究、构建差异化产品，而不必看大厂脸色。这一理念与编程民主化的精神一脉相承。

## 2026年生态现状

2025年是开源 LLM 追平闭源的转折点，到2026年，问题已经从"开源能不能用"变成了"什么场景该用开源"：

- **Meta LLaMA 4**：Scout（109B，10M上下文）和 Maverick（400B）两个版本，采用MoE架构，每次查询仅激活17B参数
- **Mistral**：Mixtral MoE架构，被大量初创公司用于私有化部署
- **[[DeepSeek]]**：DeepSeek-R1 成为中国开源AI的代表性项目
- **vLLM**：高性能推理引擎，让开源模型的部署效率逼近商业API
- **OpenClaw**：302,000+ GitHub星标，60天内成为GitHub历史上增长最快的开源项目

## 开源 vs 闭源的张力

| 维度 | 开源优势 | 闭源优势 |
|------|---------|---------|
| 成本 | 自建部署，边际成本趋零 | 按量付费，无需运维 |
| 隐私 | 数据完全本地化 | 依赖供应商承诺 |
| 定制 | 可微调、可魔改 | 开箱即用、优化到位 |
| 前沿能力 | 复杂推理仍有差距 | OpenAI o1 级推理领先 |
| 安全 | 社区审计但参差不齐 | 重金投入RLHF和安全对齐 |

关键洞察：未来不属于某一个模型，而属于**足够灵活、可以使用任何模型的架构**——这正是 [[OpenClaw 是什么|OpenClaw]] 的 [[模型无关架构]] 设计理念。

## OpenClaw 的标杆地位

OpenClaw 之所以被视为开源AI Agent的标杆，不仅因为星标数，更因为它示范了一种完整范式：
- [[MIT License]] 授权，任何人可商用
- TypeScript 编写，前端开发者零门槛
- 模型无关，从 GPT-4o 到本地 Ollama 都能跑
- 社区驱动的 Skills 生态，165+创业团队围绕它构建产品（详见商业化路径）
